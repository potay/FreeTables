FreeTables

By Paul Chun & Mark Fernandez


What's our current progress?

We decided to develop the project in C++, as it allowed us to create more robust data structures in order to keep the code tidy and readable, whilst allowing the implementation to be efficient and clean. We created the basic foundation for the code base, including a customizable testing framework in order to measure the performance of our project against other simpler and more naive implementations. We also implemented a simple global lock linked-list data structure to test our lock-free linked list against.

After completing the basic foundation in the first half of the first week, we moved on to implement the lock-free implementation of our linked-list, which would allow us to implement a lock-free hash table. We are currently trying to implement the lock-free implementation of our linked-list but have unfortunately run into some issues with our atomic structure, which allows us to do atomic compare and swaps. We have spent most of our time trying to fix this issue.

The main issue is suspected to be due to the size of the structure, which is 128 bits in size and needs to be aligned on a double word boundary. This was determined after removing the tag (which is necessary to prevent the ABA problem in lock free data structures), as it caused the program to work sequentially, but not concurrently (due to the ABA problem).

As such, this has caused us to fall behind schedule. We will be continuing to work hard on this and have thus adjusted our goals and deliverables to create a clear and practical roadmap for us for the rest of this project.

We plan to present our performance findings based on our test harness during the final competition demo and presentation and compare this performance to other standard data structure implementations. These findings will probably be illustrated in a form of a graph and other data visualizations.

What are our revised goals and deliverables?

What do we plan to achieve?
  1. Implement a correct, efficient CAS-based lock-free list that supports insertion and deletion
  2. Implement a correct, efficient lock-free working hash table that supports insertion, deletion and lookup based on the lock-free linked list.
  3. Test and measure its performance versus a hash table implemented using single grained and fine-grained locks varying thread counts.

What are we tentatively going to present during demo day?

We plan to show a series of graphs of the time taken to complete a workload versus the number of threads used for a lock-free, global locked and fine-grained implementation.

The workloads that we plan on using are

  1. 5% Insert, 5% Delete, 90% Search
  2. 10% Insert, 10% Delete, 80% Search
  3. ⅓ Insert, ⅓ Delete, ⅓ Search
